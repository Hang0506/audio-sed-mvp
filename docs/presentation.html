Tại sao công nghệ này KHÔNG lỗi thời?

❌ Cách cũ (2018–2020)

  • Rule-based: ngưỡng energy cố định
  • SVM + handcrafted features
  • Chạy trên server GPU lớn
  • Latency cao (2–5s)
  • Không generalize tốt

✅ Cách mới (MVP này)

  • YAMNet: pre-trained trên AudioSet (2M+ clips, 521 classes) — Google vẫn maintain
  • ONNX Runtime: chạy CPU, ~47ms/file — deploy edge device được
  • FireRedVAD (2024): VAD mới nhất, tốt hơn WebRTC VAD
  • Transfer learning ready: fine-tune thêm data Việt Nam
  • Scalable: thêm class mới không cần retrain toàn bộ

🗺️ Lộ trình phát triển

Phase 1 — MVP ✅ (Hiện tại)
Web app demo đầy đủ: thu âm → AI detect → khảo sát → khuyến nghị thuốc. Chạy trên browser, không cần cài app.
Phase 2 — Pilot (2–3 tháng)
Deploy 5-10 nhà thuốc thử nghiệm. Thu thập data thật từ khách Việt Nam. Fine-tune model. Đo conversion rate.
Phase 3 — Tích hợp (3–6 tháng)
Tích hợp vào app Long Châu (mobile). API cho POS quầy. Dashboard analytics cho operation team.
Phase 4 — Scale (6–12 tháng)
Rollout toàn chuỗi. Thêm classes mới (tiếng tim, tiếng thở trẻ sơ sinh). Partnership với bệnh viện cho telehealth.
