Backend Design Decisions

Storage

MVP uses JSON files (backend/storage/) — no external DB. Acceptable for 200-500 beta users. Migrate to PostgreSQL/SQLite for Phase 2.

Mock Mode Strategy

Every external dependency has graceful mock fallback:

ServiceEnv VarMock Behavior
OpenWeatherMapOPENWEATHERMAP_API_KEYReturns HCM defaults (PM2.5=85, 32°C, 78%)
Firebase FCMFIREBASE_CREDENTIALS_PATHLogs alert, returns success
YOLOv10 Food(model file existence)Returns 1-3 random Vietnamese foods
Telegram BotTELEGRAM_BOT_TOKENLogs push intent only

Rule Engine

JSON-configurable rules at backend/config/rules.json. Supports operators: gt, lt, gte, lte, eq, contains. No code deploy needed to add rules.

Health Risk Engine

Cross-references detected foods against user disease_tags + purine_db.json. Alerts: Gout+purine, Diabetes+sugar, Hypertension+salt, ENT+spicy.

iOS Design Decisions

Architecture

MVVM-lite with @State/@StateObject. NavigationStack (iOS 16+). Dark mode only for MVP.

Design System

11 color tokens, 4 font sizes, 5 spacing values. All from DesignSystem.swift. Reusable components: CircularGauge, AlertCard, ContextTile, BadgeTag, TaskCheckRow.

Audio Recording

MVP: Timer-based mock (5s countdown). Phase 2: AVAudioRecorder with 16kHz mono PCM → WAV → POST /api/analyze.

Food Camera

MVP: Mock capture (1s delay). Phase 2: AVCaptureSession → JPEG → POST /api/food-scan.
